# DB Schema Recovery Runbook — nailscosmetics.{lv,no,lt}

**Created:** 2026-05-02
**Trigger:** Boot crash `Table 'X.system_settings' doesn't exist` OR `Table 'X.lovata_shopaholic_offers' doesn't exist` on backend pages.
**Root cause:** Migration ledger (`system_plugin_history` + `system_plugin_versions`) records all plugin migrations as applied, but actual schema is partially missing (5+ tables / ~12 columns absent). Backups mirror the corrupt state — restore is **not** an option.
**Symptom:** `php artisan october:migrate` returns "Nothing to migrate" because ledger lies.
**Surgery:** clear stale history rows + reset version stamps + manual SQL for boundary-version migrations.

This runbook was executed on `.lv` (`nc_app_db`) on **2026-05-02 10:12 UTC**. Result: 5 missing tables recreated, 12 missing columns added, 9 plugin versions advanced to current. Backup at `/home/forge/db_backups/recovery/nc_pre_recovery_20260502_101203.sql.gz` (6.9M).

Apply same procedure to `.no` and `.lt` if they exhibit identical symptom.

## Phase 2 — Boot unblocker: create `system_settings` (only if missing)

October's `Renatio.FormBuilder` plugin queries `system_settings` on every boot. Missing table = `artisan` itself crashes. Create the table per migrations 5 + 27 + 34 combined schema:

```bash
mysql -h "$DB_HOST" -u "$DB_USER" -p"$DB_PWD" "$DB_NAME" <<'SQL'

  id INT UNSIGNED NOT NULL AUTO_INCREMENT,

  value MEDIUMTEXT NULL,
  site_id INT UNSIGNED NULL,
  site_root_id INT UNSIGNED NULL,
  site_group_id INT UNSIGNED NULL,
  PRIMARY KEY (id),
  INDEX system_settings_item_index (item)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
SQL
```

Test:

## Phase 4 — Manually create v1.0.0 boundary tables (offers + products)

October's `october:migrate` runs migrations from `(installed_version, current_version]`. Setting installed = 1.0.0 means v1.0.0 is **excluded**. So `create_table_offers.php` and `create_table_products.php` (both at Shopaholic v1.0.0) won't auto-rerun. Apply them directly + record in history.

```bash
mysql -h "$DB_HOST" -u "$DB_USER" -p"$DB_PWD" "$DB_NAME" <<'SQL'
START TRANSACTION;

CREATE TABLE IF NOT EXISTS lovata_shopaholic_products (
  id INT UNSIGNED NOT NULL AUTO_INCREMENT,
  active TINYINT(1) NOT NULL DEFAULT 0,
  name VARCHAR(255) NOT NULL,
  slug VARCHAR(255) NOT NULL,
  brand_id INT UNSIGNED NULL,
  category_id INT UNSIGNED NULL,
  external_id VARCHAR(255) NULL,
  code VARCHAR(255) NULL,
  preview_text TEXT NULL,
  description TEXT NULL,
  deleted_at TIMESTAMP NULL,
  created_at TIMESTAMP NULL,
  updated_at TIMESTAMP NULL,
  PRIMARY KEY (id),
  UNIQUE INDEX lovata_shopaholic_products_slug_unique (slug),
  INDEX lovata_shopaholic_products_name_index (name),
  INDEX lovata_shopaholic_products_code_index (code),
  INDEX lovata_shopaholic_products_external_id_index (external_id),
  INDEX lovata_shopaholic_products_brand_id_index (brand_id),
  INDEX lovata_shopaholic_products_category_id_index (category_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE IF NOT EXISTS lovata_shopaholic_offers (
  id INT UNSIGNED NOT NULL AUTO_INCREMENT,
  active TINYINT(1) NOT NULL DEFAULT 0,
  product_id INT UNSIGNED NULL,
  name VARCHAR(255) NOT NULL,
  code VARCHAR(255) NULL,
  external_id VARCHAR(255) NULL,
  price DECIMAL(15,2) NULL,
  old_price DECIMAL(15,2) NULL,
  quantity INT UNSIGNED NOT NULL DEFAULT 0,
  preview_text TEXT NULL,
  description TEXT NULL,
  deleted_at TIMESTAMP NULL,
  created_at TIMESTAMP NULL,
  updated_at TIMESTAMP NULL,
  PRIMARY KEY (id),
  INDEX lovata_shopaholic_offers_name_index (name),
  INDEX lovata_shopaholic_offers_code_index (code),
  INDEX lovata_shopaholic_offers_external_id_index (external_id),
  INDEX lovata_shopaholic_offers_product_id_index (product_id),
  INDEX lovata_shopaholic_offers_price_index (price),
  INDEX lovata_shopaholic_offers_old_price_index (old_price),
  INDEX lovata_shopaholic_offers_quantity_index (quantity)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

INSERT INTO system_plugin_history (code, type, version, detail, created_at)
VALUES
  ('Lovata.Shopaholic', 'script', '1.0.0', 'create_table_products.php', NOW()),
  ('Lovata.Shopaholic', 'script', '1.0.0', 'create_table_offers.php', NOW());

COMMIT;
SQL
```

## Phase 5 — Run october:migrate (~1 min)

```bash
cd "$PROJECT_DIR"
php artisan october:migrate --force 2>&1 | tail -50
```

This re-applies every migration in `(1.0.0, current]` for the 9 reset plugins. All migrations are guarded with `Schema::hasTable()` / `Schema::hasColumn()` — idempotent. Existing tables/columns untouched. Missing tables/columns created.

Expected output: list of plugin names with no errors, OR specific `vN.x.y: Migration "X.php" failed` line.

**If a migration fails:** read the error, manually create the missing table or column from the relevant migration file under `plugins/<vendor>/<plugin>/updates/`, then add a `system_plugin_history` row matching what the migration would have inserted, then re-run `php artisan october:migrate --force`. Repeat until clean.

## Phase 6 — Apply remaining v1.0.0 boundary ALTERs (cross-plugin column adds)

Same boundary issue as Phase 4: 8 ALTER migrations at v1.0.0 of their own plugin (Discounts, Subscriptions, Search, Popularity, Reviews) won't re-run because v1.0.0 is excluded. Apply manually:

```bash
mysql -h "$DB_HOST" -u "$DB_USER" -p"$DB_PWD" "$DB_NAME" <<'SQL'
START TRANSACTION;

-- Discounts v1.0.0 → offers
ALTER TABLE lovata_shopaholic_offers
  ADD COLUMN discount_id INT NULL,
  ADD COLUMN discount_value FLOAT UNSIGNED NULL,
  ADD COLUMN discount_type VARCHAR(255) NULL;

-- Subscriptions v1.0.0 → offers
ALTER TABLE lovata_shopaholic_offers
  ADD COLUMN subscription_period_id INT NULL;

-- Search v1.0.0 → products
ALTER TABLE lovata_shopaholic_products
  ADD COLUMN search_synonym TEXT NULL,
  ADD COLUMN search_content TEXT NULL;

-- Popularity v1.0.0 → products
ALTER TABLE lovata_shopaholic_products
  ADD COLUMN popularity INT NOT NULL DEFAULT 0;

-- Reviews v1.0.0 → products
ALTER TABLE lovata_shopaholic_products
  ADD COLUMN rating_data TEXT NULL,
  ADD COLUMN rating DECIMAL(5,2) NULL;

-- Subscriptions v1.0.0 → products
ALTER TABLE lovata_shopaholic_products
  ADD COLUMN is_subscription TINYINT(1) NOT NULL DEFAULT 0,
  ADD INDEX lovata_shopaholic_products_is_subscription_index (is_subscription);

-- History rows so future october:migrate doesn't try
INSERT INTO system_plugin_history (code, type, version, detail, created_at) VALUES

  ('Lovata.SubscriptionsShopaholic', 'script', '1.0.0', 'update_table_offer_add_subscription_fields.php', NOW()),
  ('Lovata.SearchShopaholic',        'script', '1.0.0', 'update_table_product.php', NOW()),
  ('Lovata.PopularityShopaholic',    'script', '1.0.0', 'update_table_product.php', NOW()),
  ('Lovata.ReviewsShopaholic',       'script', '1.0.0', 'update_table_products.php', NOW());

COMMIT;
SQL
```
