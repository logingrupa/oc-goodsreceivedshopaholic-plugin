<objective>
Build the SettingsAccessor singleton (APPLY-09) and the DRY-grep enforcement (QA-09) that all Phase 3 Apply services read settings through. Wire the accessor into the test base's tearDown hook (D-03) so cross-test bleed is impossible. Add the Makefile target plus its Pest-mirror so the invariant lives in two places (CI lint + test gate) — both fail on the same drift.

Purpose: All Phase 3 services (StockApply, ActiveFlag, InitialReset) need a single, memoized Settings accessor. Scattered `Settings::get()` calls would (1) recompute boolean conversion on every call inside a 200-line apply loop, (2) make multisite-isolation behavior hard to reason about, (3) make the QA-09 contract un-enforceable. Doing this FIRST in Wave 1 means all downstream Apply plans build against a stable contract.

Output: One service class + 2 tests + 1 Makefile target + 1 line added to TestCase. ~120 LoC total.

       ```php
       <?php
       declare(strict_types=1);
       namespace Logingrupa\GoodsReceivedShopaholic\Classes\Support;
       use Logingrupa\GoodsReceivedShopaholic\Models\Settings;

       /**
        * Single accessor for all plugin Settings reads (APPLY-09 / D-01).
        * Memoized per-request via static $arCache.
        * QA-09 CI grep gate enforces this is the SOLE caller of Settings::get(.
        * Cross-test bleed prevented via flush() called from
        * GoodsReceivedTestCase::flushPluginSingletons().
        */
       final class SettingsAccessor
       {
           /** @var array<string, bool>|null */
           private static ?array $arCache = null;

           private const KEY_ENABLED = 'enabled';
           private const KEY_AUTO_DEACTIVATE = 'auto_deactivate_on_zero';
           private const KEY_AUTO_ACTIVATE = 'auto_activate_on_stock';
           private const KEY_ALLOW_RESET = 'allow_initial_reset';

           public static function isEnabled(): bool
           {
               return self::get(self::KEY_ENABLED);
           }

           public static function autoDeactivateOnZero(): bool
           {
               return self::get(self::KEY_AUTO_DEACTIVATE);
           }

           public static function autoActivateOnStock(): bool
           {
               return self::get(self::KEY_AUTO_ACTIVATE);
           }

           public static function allowInitialReset(): bool
           {
               return self::get(self::KEY_ALLOW_RESET);
           }

           public static function flush(): void
           {
               self::$arCache = null;
           }

           private static function get(string $sKey): bool
           {
               self::$arCache ??= [
                   self::KEY_ENABLED => (bool) Settings::get(self::KEY_ENABLED),
                   self::KEY_AUTO_DEACTIVATE => (bool) Settings::get(self::KEY_AUTO_DEACTIVATE),
                   self::KEY_AUTO_ACTIVATE => (bool) Settings::get(self::KEY_AUTO_ACTIVATE),
                   self::KEY_ALLOW_RESET => (bool) Settings::get(self::KEY_ALLOW_RESET),
               ];
               return self::$arCache[$sKey];
           }
       }
       ```
       NOTE: this file is the SOLE place `Settings::get(` may appear. The QA-09 grep gate (Task 3) excludes only this path.
