<objective>
Build InitialResetService — the one-shot baseline reset (APPLY-05 / QA-06). Used ONCE per site to zero all offer quantities and deactivate all products+offers BEFORE the first goods-received import. Per D-17..D-20, the operation is gated by both a Settings toggle AND a one-shot DB guard (any prior `initial_reset_applied=true` invoice blocks future resets), snapshots every offer's prior state to `logingrupa_goods_received_initial_reset_snapshot` BEFORE writing, and uses chunked iteration (Offer::chunk(500), Product::chunk(500)) so memory stays bounded regardless of catalog size.

Purpose: Site operators run one initial reset on day-zero so the plugin owns ALL stock state from that point forward. Without the snapshot, an accidental reset is unrecoverable. Without the one-shot guard, an operator could repeatedly reset and lose intermediate apply history. Without chunked iteration, a 50,000-offer catalog OOMs the PHP-FPM worker.

Output: One service class + one test file with 5 dedicated QA-06 cases.
</objective>

       ```php
       <?php
       declare(strict_types=1);
       namespace Logingrupa\GoodsReceivedShopaholic\Classes\Apply;

       use Carbon\Carbon;
       use Logingrupa\GoodsReceivedShopaholic\Classes\Support\SettingsAccessor;
       use Logingrupa\GoodsReceivedShopaholic\Classes\Exception\InitialResetNotAllowedException;
       use Logingrupa\GoodsReceivedShopaholic\Models\InitialResetSnapshot;
       use Logingrupa\GoodsReceivedShopaholic\Models\Invoice;
       use Lovata\Shopaholic\Models\Offer;
       use Lovata\Shopaholic\Models\Product;
       use October\Rain\Database\Collection as DbCollection;

       /**
        * One-shot baseline reset (APPLY-05 / QA-06).
        *
        * Pre-checks (D-17):
        *   - Settings.allow_initial_reset MUST be true
        *   - No prior Invoice with initial_reset_applied=true MUST exist
        *
        * Operation (D-18, D-19):
        *   1) Snapshot every offer + its product to
        *      logingrupa_goods_received_initial_reset_snapshot via BATCHED insert
        *      (Offer::chunk(500) + InitialResetSnapshot::insert([…rows…]))
        *   2) Zero offers: chunk 500 → quantity=0, active=false, managed_by='plugin' (saveQuietly)
        *   3) Deactivate products: chunk 500 → active=false (saveQuietly)
        *   4) Mark Invoice.initial_reset_applied=true (saveQuietly)
        *
        * Rollback story (D-20): snapshot is the rollback artifact. This service
        * ships only the WRITE path. A future ops command (Phase 5 runbook) walks
        * snapshot rows in reverse to restore prior state.
        *
        * Caller controls DB::transaction boundary — this service does NOT wrap
        * in transaction itself, so a caller can compose with other writes.
        */
       final class InitialResetService
       {
           private const CHUNK_SIZE = 500;
           private const PROVENANCE_PLUGIN = 'plugin';

           public function reset(Invoice $obInvoice): void
           {
               $this->assertAllowed();

               $iInvoiceId = (int) $obInvoice->id;
               $this->snapshotAllOffers($iInvoiceId);
               $this->zeroAllOffers();
               $this->deactivateAllProducts();
               $this->markInvoiceReset($obInvoice);
           }

           /** @throws InitialResetNotAllowedException */
           private function assertAllowed(): void
           {
               if (! SettingsAccessor::allowInitialReset()) {
                   throw new InitialResetNotAllowedException(
                       (string) \Lang::get('logingrupa.goodsreceivedshopaholic::lang.exception.initial_reset_not_allowed'),
                       ['reason' => 'settings_disabled'],
                   );
               }

               if (Invoice::where('initial_reset_applied', true)->exists()) {
                   throw new InitialResetNotAllowedException(
                       (string) \Lang::get('logingrupa.goodsreceivedshopaholic::lang.exception.initial_reset_not_allowed'),
                       ['reason' => 'already_applied'],
                   );
               }
           }

           /** Snapshot every offer's prior state. Returns count of rows inserted. */
           private function snapshotAllOffers(int $iInvoiceId): int
           {
               $iCount = 0;
               $sNow = Carbon::now()->toDateTimeString();
               Offer::with('product:id,active')
                   ->chunkById(self::CHUNK_SIZE, function (DbCollection $obChunk) use ($iInvoiceId, $sNow, &$iCount): void {
                       $arRows = [];
                       foreach ($obChunk as $obOffer) {
                           $obProduct = $obOffer->product;  // eager-loaded above
                           $arRows[] = [
                               'invoice_id' => $iInvoiceId,
                               'offer_id' => (int) $obOffer->id,
                               'prior_quantity' => (int) $obOffer->quantity,
                               'prior_offer_active' => (bool) $obOffer->active,
                               'prior_product_id' => $obProduct !== null ? (int) $obProduct->id : null,
                               'prior_product_active' => $obProduct !== null ? (bool) $obProduct->active : false,
                               'created_at' => $sNow,
                           ];
                       }
                       if ($arRows !== []) {
                           InitialResetSnapshot::insert($arRows);
                           $iCount += count($arRows);
                       }
                   });
               return $iCount;
           }

           /** Zero every offer's quantity + deactivate. Returns count touched. */
           private function zeroAllOffers(): int
           {
               $iCount = 0;
               Offer::chunkById(self::CHUNK_SIZE, function (DbCollection $obChunk) use (&$iCount): void {
                   foreach ($obChunk as $obOffer) {
                       $obOffer->quantity = 0;
                       $obOffer->active = false;
                       $obOffer->active_managed_by = self::PROVENANCE_PLUGIN;
                       $obOffer->saveQuietly();
                       $iCount++;
                   }
               });
               return $iCount;
           }

           /** Deactivate every product. Returns count touched. */
           private function deactivateAllProducts(): int
           {
               $iCount = 0;
               Product::chunkById(self::CHUNK_SIZE, function (DbCollection $obChunk) use (&$iCount): void {
                   foreach ($obChunk as $obProduct) {
                       $obProduct->active = false;
                       $obProduct->saveQuietly();
                       $iCount++;
                   }
               });
               return $iCount;
           }

           private function markInvoiceReset(Invoice $obInvoice): void
           {
               $obInvoice->initial_reset_applied = true;
               $obInvoice->saveQuietly();
           }
       }
       ```

|----------|-------------|
| Operator → InitialResetService | Once-per-site privileged operation; double guard (Settings + DB one-shot check) prevents accidental double-reset |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05-01 | Tampering | Repeated reset destroys apply history | mitigate | Two-gate guard: SettingsAccessor::allowInitialReset() + Invoice::where('initial_reset_applied', true)->exists(). EITHER blocks future resets. Asserted by RequiresAllowInitialResetSetting + OneShotEnforced tests |
| T-03-05-02 | Information disclosure | Lost prior state on accidental reset | mitigate | EVERY offer's prior state is snapshotted to logingrupa_goods_received_initial_reset_snapshot BEFORE any write. Asserted by SnapshotsBeforeWriteTest. Rollback feasibility asserted by RollbackRestoresExactPriorStateTest |
| T-03-05-03 | Denial of service | OOM on 50,000-offer catalog | mitigate | chunkById(500) — bounded memory regardless of catalog size. Asserted by ChunkedNotSingleStatementTest (1500 offers → 3+ chunk pages) |
| T-03-05-04 | Tampering | DB::statement bypass of cache hooks | mitigate | NEVER uses DB::statement. saveQuietly per row + batched InitialResetSnapshot::insert (the snapshot table has NO model handlers, so insert is safe). Verified by post-write grep |
| T-03-05-05 | Repudiation | Lost audit trail on reset | mitigate | Phase 3 plan 03-07 ApplyOrchestrator (or Phase 4 controller for direct reset path) wraps reset() with ImportAuditService::logInitialReset(invoice_id, offers_zeroed, products_deactivated).
