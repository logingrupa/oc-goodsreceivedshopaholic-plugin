    2) Create `classes/support/ImportAuditService.php`:
       ```php
       <?php
       declare(strict_types=1);
       namespace Logingrupa\GoodsReceivedShopaholic\Classes\Support;

       use Illuminate\Support\Facades\Log;
       use Illuminate\Support\Str;
       use Logingrupa\GoodsReceivedShopaholic\Classes\Dto\ApplyResult;
       use Logingrupa\GoodsReceivedShopaholic\Classes\Dto\ParsedInvoice;

       /**
        * Vendor-inlined audit log service for goods-received imports (APPLY-10 / D-04).
        * Routes structured-context log entries through Laravel's Log facade so the
        * host project's configured channels (file / stack / sentry) pick them up.
        *
        * NO soft-dep on Logingrupa.ExtendShopaholic per D-14 — this class is the
        * canonical audit boundary for the plugin. ~50-80 LoC ceiling enforced by
        * keeping log methods one-liners that delegate to a private build helper.
        *
        * Threat model:
        *   - T-03-02-01 (log injection): context arrays pass through Laravel's
        *     PSR-3 formatter which json_encodes them — control chars become
        *     literal escape sequences. No raw user input concatenated into the
        *     message string. Mitigation inherited.
        *   - T-03-02-02 (PII leak): NO email / name / raw HTM body fields are
        *     ever logged. Only integer invoice_id / offer_id and known-safe
        *     enum strings (event, reason, source_filename).
        */
       final class ImportAuditService
       {
           public function logApply(int $iInvoiceId, ApplyResult $obResult, int $iAppliedByUserId): void
           {
               Log::info('goodsreceived.apply', $this->buildApplyContext($iInvoiceId, $obResult, $iAppliedByUserId));
           }

           public function logParse(int $iInvoiceId, ParsedInvoice $obParsed): void
           {
               Log::info('goodsreceived.parse', $this->buildParseContext($iInvoiceId, $obParsed));
           }

           /** @param array<string, mixed> $arContext */
           public function logReject(string $sReason, array $arContext = []): void
           {
               Log::warning('goodsreceived.reject', array_merge([
                   'event' => 'reject',
                   'reason' => $sReason,
                   'correlation_id' => $this->correlationId(),
               ], $arContext));
           }

           public function logInitialReset(int $iInvoiceId, int $iOffersZeroed, int $iProductsDeactivated): void
           {
               Log::info('goodsreceived.initial_reset', [
                   'event' => 'initial_reset',
                   'invoice_id' => $iInvoiceId,
                   'offers_zeroed' => $iOffersZeroed,
                   'products_deactivated' => $iProductsDeactivated,
                   'correlation_id' => $this->correlationId(),
               ]);
           }

           /** @return array<string, mixed> */
           private function buildApplyContext(int $iInvoiceId, ApplyResult $obResult, int $iAppliedByUserId): array
           {
               return [
                   'event' => 'apply',
                   'invoice_id' => $iInvoiceId,
                   'units_added' => $obResult->units_added,
                   'offers_touched' => $obResult->offers_touched,
                   'lines_applied' => $obResult->lines_applied,
                   'lines_skipped' => $obResult->lines_skipped,
                   'applied_by' => $iAppliedByUserId,
                   'correlation_id' => $this->correlationId(),
               ];
           }

           /** @return array<string, mixed> */
           private function buildParseContext(int $iInvoiceId, ParsedInvoice $obParsed): array
           {
               return [
                   'event' => 'parse',
                   'invoice_id' => $iInvoiceId,
                   'invoice_number' => $obParsed->invoice_number,
                   'source_filename' => $obParsed->source_filename,
                   'total_lines' => count($obParsed->lines),
                   'skipped_count' => count($obParsed->skipped_rows),
                   'correlation_id' => $this->correlationId(),
               ];
           }

           private function correlationId(): string
           {
               // Laravel 12 ships uuid7. Fallback to uuid() (v4) handled at runtime
               // because both return Stringable Uuid; toString() is canonical.
               if (method_exists(Str::class, 'uuid7')) {
                   return (string) Str::uuid7();
               }
               return (string) Str::uuid();
           }
       }
       ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Plugin code → Laravel Log facade | Audit context flows out to log files; control chars in user-controlled strings (e.g., `source_filename`) must not forge fake log lines |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | Tampering | Log injection via attacker-controlled invoice_number / source_filename | mitigate | Laravel's PSR-3 formatter json_encodes the context array; control chars (`\n`, `\r`, `\t`) become literal escape sequences. The `$sMessage` parameter is always a fixed string literal — never concatenated user input |
| T-03-02-02 | Information disclosure | PII in logs | mitigate | Service interface accepts only INTEGER ids (invoice_id, offer_id, applied_by_user_id) + ENUM strings (event, reason). NEVER accepts email / name / raw HTM body. Method signatures enforce this at compile time |
| T-03-02-03 | Repudiation | Audit log tampering | accept | Logs are write-only via Laravel's Log facade; integrity protection is the host's responsibility (logrotate immutable mode, sentry, etc.). Out of scope per D-14 (vendor-inlined ~50-80 LoC) |
| T-03-02-04 | Repudiation | Missing correlation between parse + apply log lines | mitigate | Each log call gets a fresh correlation_id; ApplyOrchestrator (plan 03-07) MUST pass the same correlation_id through if joining parse+apply is needed. NOTE: this plan does NOT pre-thread that — orchestrators get fresh ids per call. If join is required, a future enhancement adds a constructor-injected correlation_id; deferring for now per ~50-80 LoC ceiling |
</threat_model>
