<objective>
Build the StockApplyService — the side-effect-isolated heart of Phase 3. Increments `lovata_shopaholic_offers.quantity` per matched line via `Eloquent saveQuietly()` (NOT `DB::statement`, NOT `->save()`), then exposes a separate `flushAffectedCaches(array<int> $arOfferIds)` method the orchestrator (plan 03-07) calls AFTER the DB transaction commits.

The "saveQuietly + post-commit batched flush" contract is the ENTIRE point of APPLY-01/02/QA-04: a 200-line apply should hit the offers table 200 times (one update per matched offer, but offers are de-duplicated by id) and the cache stores 4-5 times TOTAL — not 1200+ times.

Purpose: Per D-07, calling `$obOffer->save()` inside a 200-line loop fires Lovata's `OfferModelHandler::afterSave` once per save → which calls `OfferListStore::instance()->active->clear() + sorting->clear(SORT_*) + product_active list clear + product_item cache clear`. That's 8-12 cache flushes per save × 200 lines = 1600-2400 cache flushes per apply. `saveQuietly()` skips the bound events entirely. Then ONE manual batched flush per store at the end gives correctness with O(stores) flushes instead of O(lines × stores).

Output: One service class + two tests. The service is the load-bearing engine of Phase 3 — every other Wave 2/3 plan depends on it (directly or transitively).
</objective>

       ```php
       <?php
       declare(strict_types=1);
       namespace Logingrupa\GoodsReceivedShopaholic\Classes\Apply;

       use Carbon\Carbon;
       use Logingrupa\GoodsReceivedShopaholic\Classes\Dto\ApplyResult;
       use Logingrupa\GoodsReceivedShopaholic\Models\Invoice;
       use Logingrupa\GoodsReceivedShopaholic\Models\InvoiceLine;
       use Lovata\Shopaholic\Classes\Item\OfferItem;
       use Lovata\Shopaholic\Classes\Store\OfferListStore;
       use Lovata\Shopaholic\Classes\Store\ProductListStore;
       use Lovata\Shopaholic\Models\Offer;
       use October\Rain\Database\Collection as DbCollection;

       /**
        * Stock writer for the goods-received import pipeline (APPLY-01 / APPLY-02 / QA-04).
        *
        * Contract:
        *   - apply(): mutates offer.quantity via saveQuietly so Lovata's
        *     OfferModelHandler::afterSave is NOT bound — eliminating the per-save
        *     8-12 cache-flush cascade. Returns StockApplyOutcome.
        *   - flushAffectedCaches(): the orchestrator (plan 03-07) calls this AFTER
        *     DB::transaction commits (D-10). One flush per affected store, NOT
        *     N×lines.
        *
        * NEVER uses DB::statement (skips cache hooks AND attribute casting).
        * NEVER uses ->save() per line (8-12 flushes per offer × N lines = QA-04 anti-pattern).
        */
       final class StockApplyService
       {
           public function apply(Invoice $obInvoice, DbCollection $obMatchedLines): StockApplyOutcome
           {
               $arOfferDeltas = $this->groupLinesByOffer($obMatchedLines);

               $iLinesApplied = 0;
               $iLinesSkipped = 0;
               $iUnitsAdded = 0;
               $arAffectedOfferIds = [];

               // Pass 1: write offer stock (one save per UNIQUE offer).
               foreach ($arOfferDeltas as $iOfferId => $iDelta) {
                   $bSaved = $this->writeOfferStock($iOfferId, $iDelta);
                   if (! $bSaved) { continue; }
                   $arAffectedOfferIds[] = $iOfferId;
                   $iUnitsAdded += $iDelta;
               }

               // Pass 2: mark applied=true per LINE (not per offer; multiple lines may
               // share an offer — each line is a distinct audit row).
               foreach ($obMatchedLines as $obLine) {
                   if ($obLine->matched_offer_id === null) { $iLinesSkipped++; continue; }
                   if (! in_array($obLine->matched_offer_id, $arAffectedOfferIds, true)) {
                       $iLinesSkipped++; continue;
                   }
                   $this->markLineApplied($obLine);
                   $iLinesApplied++;
               }

               return new StockApplyOutcome(
                   result: new ApplyResult(
                       units_added: $iUnitsAdded,
                       offers_touched: count($arAffectedOfferIds),
                       lines_applied: $iLinesApplied,
                       lines_skipped: $iLinesSkipped,
                   ),
                   affected_offer_ids: array_values($arAffectedOfferIds),
               );
           }

           /**
            * After-commit batched cache flush. Called by ApplyOrchestrator AFTER
            * DB::transaction commits (D-10). One flush per affected store.
            *
            * @param list<int> $arOfferIds
            */
           public function flushAffectedCaches(array $arOfferIds): void
           {
               // List-level stores: flush once each (no per-id loop needed).
               OfferListStore::instance()->active->clear();
               OfferListStore::instance()->sorting->clear(OfferListStore::SORT_NO);
               OfferListStore::instance()->sorting->clear(OfferListStore::SORT_NEW);
               ProductListStore::instance()->active->clear();

               // Item-level cache: flush once per affected offer id (this loop is
               // O(unique_offers), NOT O(lines), so it stays bounded).
               foreach ($arOfferIds as $iOfferId) {
                   OfferItem::clearCache($iOfferId);
               }
           }

           /**
            * Group matched lines by offer id; sum qty (or override_qty when set).
            *
            * @return array<int, int>  map<offer_id, total_qty_delta>
            */
           private function groupLinesByOffer(DbCollection $obMatchedLines): array
           {
               $arDeltas = [];
               foreach ($obMatchedLines as $obLine) {
                   if ($obLine->matched_offer_id === null) { continue; }
                   $iOfferId = (int) $obLine->matched_offer_id;
                   $iLineQty = $this->effectiveQty($obLine);
                   $arDeltas[$iOfferId] = ($arDeltas[$iOfferId] ?? 0) + $iLineQty;
               }
               return $arDeltas;
           }

           private function effectiveQty(InvoiceLine $obLine): int
           {
               return $obLine->override_qty !== null
                   ? (int) $obLine->override_qty
                   : (int) $obLine->qty;
           }

           private function writeOfferStock(int $iOfferId, int $iDelta): bool
           {
               $obOffer = Offer::find($iOfferId);
               if ($obOffer === null) { return false; }

               // Direct attribute write — saveQuietly() is THE point. saveQuietly
               // bypasses Lovata's OfferModelHandler::afterSave, eliminating the
               // 8-12 cache flushes per save (QA-04 anti-pattern guard). The
               // post-commit batched flush in flushAffectedCaches restores cache
               // consistency at O(stores) cost instead of O(lines × stores).
               $obOffer->quantity = (int) $obOffer->quantity + $iDelta;
               $obOffer->saveQuietly();
               return true;
           }

           private function markLineApplied(InvoiceLine $obLine): void
           {
               $obLine->applied = true;
               $obLine->applied_at = Carbon::now();
               $obLine->saveQuietly();
           }
       }
       ```
