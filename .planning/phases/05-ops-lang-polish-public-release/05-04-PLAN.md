    **Step 2: Inspect phpunit.xml current shape.**

    Read `phpunit.xml` first. If a `<source>` (PHPUnit 10+) or `<coverage><include>` (legacy) block already exists with the right directories, no edit needed. If missing, ADD a `<source>` block (PHPUnit 10/11/12 native syntax — match the project's PHPUnit version):

    ```xml
    <source>
        <include>
            <directory>classes</directory>
            <directory>components</directory>
            <directory>console</directory>
            <directory>controllers</directory>
            <directory>models</directory>
            <file>Plugin.php</file>
        </include>
        <exclude>
            <directory>classes/dto</directory>  <!-- pure readonly DTOs, no logic to instrument -->
            <directory>classes/exception</directory>  <!-- typed exception classes, no logic -->
        </exclude>
    </source>
    ```

    Rationale for excludes: DTOs + exceptions are pure data shapes with constructor-only logic; instrumenting them inflates the denominator without adding signal. If the operator prefers them included, drop the `<exclude>` block — surface as a checkpoint decision.

    **Step 3: Measure baseline coverage.**

    Run a one-shot uncapped measurement to find the actual current floor:
    ```bash
    cd $(git rev-parse --show-toplevel) && vendor/bin/pest --configuration plugins/logingrupa/goodsreceivedshopaholic/phpunit.xml --coverage 2>&1 | tail -50
    ```

    Note the reported "Total" line coverage percentage. Round DOWN to the nearest 5 to get the threshold N (e.g., measured 87.3% → set N=85; measured 78.1% → set N=75). This produces a stable gate that catches regressions but tolerates harmless ±2-3% drift from test additions.

    **Step 4: Add coverage target to Makefile.**

    Use Edit tool. Add to the `.PHONY` line:
    ```
    .PHONY: test analyse baseline rector-dry rector phpmd pint pint-test lint-settings-accessor coverage all
    ```

    Add the new target after `test:` (or near other test targets):
    ```makefile
    # OPS-05: coverage gate. Threshold pinned at the measured floor minus drift tolerance
    # (D-13 pragmatic — if measured 87%, set --min=85; if 78%, set --min=75).
    # Driver: PCOV preferred (faster), Xdebug (mode=coverage) acceptable fallback.
    coverage:
    	cd $(PROJECT_ROOT) && $(VENDOR_BIN)/pest --configuration $(PLUGIN_DIR)/phpunit.xml --coverage --min=N
    ```

    Replace `N` with the measured threshold from Step 3.

    Update the `all` target to include coverage. Two acceptable patterns:

    **Option A (separate target — preferred per D-11):**
    ```makefile
    all: pint-test lint-settings-accessor analyse phpmd test coverage
    ```
    Pros: dev iteration `make test` stays fast (no coverage instrumentation overhead — usually 2-3x slowdown with PCOV, 5-10x with Xdebug). `make all` (used in CI / release gate) runs the full coverage gate.

    **Option B (fold into test):** Just add `--coverage --min=N` to the existing `test:` target. Pros: simpler. Cons: every `make test` during dev pays the coverage cost. D-11 explicitly recommends Option A — go with A unless operator overrides at checkpoint.

    MUST exit 0. All five sub-targets pass.
