<objective>
Resolve the canonical invoice number for a `.HTM` upload with two-tier fallback per D-18..D-20: body marker first (more reliable, distributor-controlled), filename second (operator-controllable but structured), throw if both fail. Resolver also emits country_code (`'no'`/`'lv'`/`'lt'`) and invoice_date parsed from the filename for the parser to embed in `ParsedInvoice` header fields without a second filename parse.

Purpose: A misnamed file should not silently inherit the wrong invoice number from filename. Body-first ordering makes accidental rename harmless; explicit throw on total miss prevents nameless invoices from being persisted.
Output: 1 resolver class + 1 unit test verified against the 3 hermetic fixtures already in `tests/fixtures/invoices/`.
</objective>

    Create file `classes/parser/InvoiceNumberResolver.php` with:

    1. Strict types + namespace `Logingrupa\GoodsReceivedShopaholic\Classes\Parser`
    2. `use Logingrupa\GoodsReceivedShopaholic\Classes\Exception\InvoiceNumberMissingException;`
    3. `final class InvoiceNumberResolver`
    4. Two private const regex patterns:
       - BODY_MARKER_REGEX: `/(?:Invoice|Faktura|Rechnung)\s*(?:No|Nr|nr)[^\w\d]*([A-Z]{0,4}\d{4,})/iu`
         - Covers EN ("Invoice No"), Norwegian ("Faktura Nr"), German ("Rechnung Nr") label variants
         - `[^\w\d]*` skips colons/periods/whitespace between label and value
         - `([A-Z]{0,4}\d{4,})` captures `PRO033328` style; greedy; `u` flag for Unicode safety
       - FILENAME_REGEX: `/^Nr_(PRO\d+)_([A-Za-z]{2,3})_(\d{8})\.HTM$/i`
         - Anchored both ends
         - Captures: (1) canonical PRO+digits, (2) country code, (3) DDMMYYYY
    5. Public static `resolve(string $sHtmlBody, string $sFilename): array` with PHPDoc per `<interfaces>` block. Body-of-method, in order:
       a. `$sBasename = basename($sFilename);`
       b. `$arFilenameMeta = self::parseFilenameMeta($sBasename);` (may be null when filename doesn't match)
       c. `$sBodyNumber = self::extractFromBody($sHtmlBody);`
       d. **Branch 1** — body hit: if `$sBodyNumber !== null`, return tuple with `resolved_via => 'body'`, country_code/invoice_date sourced from `$arFilenameMeta` if present (else null)
       e. **Branch 2** — filename hit: else if `$arFilenameMeta !== null`, return tuple with `resolved_via => 'filename'`
       f. **Branch 3** — both miss: throw InvoiceNumberMissingException with `\Lang::get('logingrupa.goodsreceivedshopaholic::lang.exception.invoice_number_missing')` and arContext `['source_filename' => $sBasename, 'tried_body' => true, 'tried_filename' => true]`
    6. Private static `extractFromBody(string $sHtmlBody): ?string`:
       a. `$sText = html_entity_decode(strip_tags($sHtmlBody), ENT_QUOTES | ENT_HTML5, 'UTF-8');`
       b. `$sText = preg_replace('/\s+/u', ' ', $sText) ?? '';` — collapse whitespace runs (including the decoded `&nbsp;`)
       c. `if (preg_match(self::BODY_MARKER_REGEX, $sText, $arMatch) === 1) return strtoupper($arMatch[1]);`
       d. return null
    7. Private static `parseFilenameMeta(string $sBasename): ?array`:
       a. `if (preg_match(self::FILENAME_REGEX, $sBasename, $arMatch) !== 1) return null;`
       b. `$obDate = \DateTimeImmutable::createFromFormat('!dmY', $arMatch[3]);`
       c. `if ($obDate === false) return null;` — malformed date → treat as filename miss
       d. return `['invoice_number' => strtoupper($arMatch[1]), 'country_code' => strtolower($arMatch[2]), 'invoice_date' => $obDate]`
       e. PHPDoc return type: `@return array{invoice_number: string, country_code: string, invoice_date: \DateTimeImmutable}|null`

    No `else` after early return (D-37). Max nesting level 1. Each function under 35 lines.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04-01 | Spoofing (silent number swap via rename) | Operator renames file to look like a different invoice | mitigate | Body-first ordering: body marker takes precedence over filename. Misnamed file with intact body still resolves correctly. |
| T-02-04-02 | Tampering (regex catastrophic backtracking) | BODY_MARKER_REGEX + huge crafted HTM | accept | Pattern uses no nested quantifiers and no overlapping alternations; PCRE worst-case is linear in input. The HTM size is bounded upstream by Phase 4 controller (UI-12 covers `upload_max_filesize`). |
| T-02-04-03 | Information Disclosure | Body content echoed in exception arContext | mitigate | arContext only carries `source_filename` (operator-supplied), no body content. Filename may carry PII if operator names files unwisely; that's an operational concern outside this resolver. |
| T-02-04-04 | Denial-of-Service | Malformed date in filename forces throw | accept | Throw is the correct outcome — operator must rename. No retry/loop possible. |
