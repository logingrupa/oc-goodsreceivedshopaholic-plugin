<objective>
Two pure stateless normalizers. `QuantityNormalizer` is the **stock-write guard** — it rejects decimal quantities (e.g., `'5,12'` from a distributor typo) BEFORE Eloquent's silent `setQuantityAttribute` int-clamp would corrupt stock to `5`. `PriceNormalizer` converts decimal-comma price strings to floats for audit display only — never written to stock.

Purpose: The single most damaging silent-corruption path in the whole import is `(int) '5,12' === 5`. This plan closes it with a hard exception at the parser boundary and a Pest test that asserts the throw.
Output: 2 normalizer classes + 2 unit tests covering QA-02 RejectsDecimalQuantityTest and the decimal-comma price round-trip.
</objective>

    **QuantityNormalizer.php**:
      1. Strict types + namespace + `use Logingrupa\GoodsReceivedShopaholic\Classes\Exception\InvalidQuantityException;`
      2. `final class QuantityNormalizer`
      3. Single public static method:
         ```php
         /**
          * @param array<string, mixed> $arContext extra forensic context to attach to thrown exception (e.g., ['row_index' => 7])
          * @throws InvalidQuantityException
          */
         public static function parseQuantity(string $sRaw, array $arContext = []): int
         {
             $sTrimmed = trim($sRaw);

             if ($sTrimmed === '' || preg_match('/^\d+$/', $sTrimmed) !== 1) {
                 throw new InvalidQuantityException(
                     \Lang::get('logingrupa.goodsreceivedshopaholic::lang.exception.invalid_quantity'),
                     array_merge(['raw' => $sRaw], $arContext),
                 );
             }

             $iQty = (int) $sTrimmed;

             if ($iQty <= 0) {
                 throw new InvalidQuantityException(
                     \Lang::get('logingrupa.goodsreceivedshopaholic::lang.exception.invalid_quantity'),
                     array_merge(['raw' => $sRaw, 'parsed' => $iQty], $arContext),
                 );
             }

             return $iQty;
         }
         ```
      4. No instance state. No private helpers. Function is 18 lines, fits the <70 line + 1-nesting-level rule (D-37).

    Note on regex `/^\d+$/`:
      - `5,12` → no match (comma fails) → throws
      - `5.12` → no match (period fails) → throws
      - `5e2` → no match (`e` fails) → throws
      - `0` → matches → falls to `<= 0` check → throws
      - `5` → matches, `(int) 5 = 5`, returns 5

    Note on max-int safety: PHP's `(int) '99999999999999999999'` will silently overflow to `PHP_INT_MAX` on 64-bit and to a negative number on 32-bit. The `<= 0` guard catches the 32-bit overflow path. Do NOT add an upper bound — distributor invoices realistically cap at qty 9999 per line; YAGNI.

    **Lang facade import**: prefer fully-qualified `\Lang::get(...)` over `use Lang;` to avoid PHPStan complaints about the October facade resolution. Both work; FQ keeps the file shorter.

    **QuantityNormalizerTest.php**: one `it()` block per case enumerated under `<behavior>`. Use Pest's `dataset` if you want compaction, but explicit `it()` blocks read better in failure output. For throw assertions:
      ```php
      it('rejects decimal-comma quantity', function (): void {
          expect(fn () => QuantityNormalizer::parseQuantity('5,12'))
              ->toThrow(InvalidQuantityException::class);
      });

      it('attaches raw value to exception context', function (): void {

              $this->fail('expected exception not thrown');
      ```

         ```php
         public static function parsePrice(?string $sRaw): ?float
         {
             if ($sRaw === null) {
                 return null;
             }

             $sTrimmed = trim($sRaw);

             if ($sTrimmed === '') {
                 return null;

             $sNormalized = str_replace(',', '.', $sTrimmed);
         ```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-01 | Tampering (silent stock corruption) | `QuantityNormalizer` against decimal-comma input | mitigate | Strict regex `/^\d+$/` rejects `'5,12'`, `'5.12'`, `'5e2'`. Throws `InvalidQuantityException` with raw value in context. Test enforces every malformed-input rejection path. |
| T-02-03-02 | Tampering (zero/negative qty) | `QuantityNormalizer` invariant | mitigate | Explicit `<= 0` check after `(int)` cast. Test asserts. |

| T-02-03-04 | Information Disclosure | Raw value echoed in exception context | accept | Raw value is a goods-received line (EAN/qty/price). Not PII. Already discussed in 02-02 threat model. |

The `'raw' => $sRaw` value lands inside an array structure that the framework serializes safely. |
