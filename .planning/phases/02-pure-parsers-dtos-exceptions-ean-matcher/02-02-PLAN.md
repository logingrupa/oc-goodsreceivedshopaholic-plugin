GoodsReceivedException (abstract base — D-07):
```php
abstract class GoodsReceivedException extends \RuntimeException {
    /**
     * @param array<string, mixed> $arContext
     */
    public function __construct(
        string $sMessage,
        public readonly array $arContext = [],
        ?\Throwable $obPrevious = null,
    ) {
        parent::__construct($sMessage, 0, $obPrevious);
    }
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Exception → log sink | `Log::error()` callers will receive `$obException->arContext` arrays whose values may originate (transitively) from attacker-controlled HTM. The exception object is the boundary between "structured data inside service" and "string written to log file". |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02-01 | Tampering | `$arContext` after construction | mitigate | Property declared `public readonly array $arContext` — PHP 8.4 enforces immutability. Test asserts write throws `\Error`. |
| T-02-02-02 | Information Disclosure | Log entries via `jsonContext` | mitigate | `JSON_UNESCAPED_SLASHES | JSON_UNESCAPED_UNICODE` keeps payload readable; default `json_encode` escapes control chars (newline, CR, tab) into `\nXX` literals — prevents log-injection (attacker forging fake log lines via embedded `\n`). Test asserts no raw `\n`/`\r` byte appears in output. |
| T-02-02-03 | Denial-of-Service | `jsonContext` against unencodable input (resources, recursive refs) | mitigate | `?: '{}'` fallback — `json_encode` returns false on failure, the `?:` returns the string `'{}'` so callers always get a string. Test asserts this. |
| T-02-02-04 | Repudiation | Exception thrown without forensic context | accept (caller responsibility) | Constructors accept `$arContext` as second arg; callers (parser, matcher, orchestrators) decide what to include. Convention documented in CLAUDE.md and plan 02-04/05/06 actions. Not enforced at exception-class level. |
</threat_model>
