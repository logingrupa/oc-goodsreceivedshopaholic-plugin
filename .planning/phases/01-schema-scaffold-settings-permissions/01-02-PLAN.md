KEY notes for October Rain v4:
- `$hasMany`, `$belongsTo`, `$hasOne`, `$morphMany` are ARRAYS, NOT methods. October's `HasRelationships` trait introspects these arrays and creates dynamic methods. This is CORRECT October convention; do NOT replace with method-based relations (would require `\October\Rain\Database\Relations\HasMany` returns).
- `$dates` array tells Carbon to cast those columns to `\Carbon\Carbon` instances.
- `$casts` is for non-date scalar casts (bool, int, float, decimal:N).
- `Validation` trait reads `$rules` and validates on save.
- `public $table`, `public $rules`, `public $fillable`, `public $casts`, `public $hasMany`, `public $belongsTo`, `public $dates` — visibility MUST be public (October trait introspection requirement).
- PHPStan level 10 will demand full `@property` PHPDoc blocks for every column. List EVERY column from the migration.

 *

        'invoice_id',
        'offer_id',
        'prior_quantity',
        'prior_offer_active',
        'prior_product_id',
        'prior_product_active',

| Boundary | Description |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-07 | Tampering | `InvoiceLine::ean` mass-assignment | mitigate | `$fillable` whitelist + `Validation` trait `$rules` enforce `ean` as `required|string|max:13`. Eloquent silently truncates strings exceeding column length on MySQL — `max:13` validation catches this BEFORE write. |
| T-01-08 | Information Disclosure | `Invoice::$casts` | accept | All casts are scalar (int, bool, decimal). No PII fields are cast to objects/arrays that would expand attack surface. |
