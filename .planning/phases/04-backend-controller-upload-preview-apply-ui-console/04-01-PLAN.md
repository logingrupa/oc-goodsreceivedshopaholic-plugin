Per D-34 + D-35, extend `Plugin.php`:

1. Add `use Illuminate\Support\Facades\App;` and `use Illuminate\Support\Facades\Log;` to the import block (preserve the existing `use Logingrupa\GoodsReceivedShopaholic\Models\Settings;` and `use System\Classes\PluginBase;`).

2. Add a private static helper `parseIniSize(string $sIni): int` to `Plugin`. Implementation contract:
   - Trim the input.
   - If empty → return 0.
   - Strip the trailing suffix character if it is one of K, M, G (case-insensitive). Multiply numeric prefix by 1024, 1024**2, 1024**3 respectively.
   - If no suffix → cast prefix to int.
   - If prefix is non-numeric (e.g. bare 'M') → return 0 (safe degradation, never throw — this is a self-check, not a validator).

3. Replace the empty `boot()` body with the self-check from D-34, but as the FIRST and ONLY thing in the body. Use `App::runningInBackend()` as a guard clause (Tiger-Style early return); if false, return immediately.

```php
public function boot(): void
{
    if (! App::runningInBackend()) {
        return;
    }

    $iMaxUploads = (int) ini_get('max_file_uploads');
    if ($iMaxUploads < 20) {
        Log::warning('GoodsReceived: max_file_uploads is below 20', [
            'current' => $iMaxUploads,
            'recommended' => 20,
        ]);
    }

    $sUploadMaxSize = (string) ini_get('upload_max_filesize');
    $iUploadMaxBytes = self::parseIniSize($sUploadMaxSize);
    if ($iUploadMaxBytes < 10 * 1024 * 1024) {
        Log::warning('GoodsReceived: upload_max_filesize is below 10M', [
            'current' => $sUploadMaxSize,
            'recommended' => '10M',
        ]);
    }
}
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| backend bootstrap | `App::runningInBackend()` gate — frontend bootstrap path skips self-check entirely (defensive; prevents log noise on every public page hit) |
| ini_get | host-controlled value, untrusted in the sense that a misconfigured server reports anomalous numbers — must NEVER throw or cast-coerce in a way that breaks boot |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-01 | Denial of Service | parseIniSize on malformed input | mitigate | parseIniSize returns 0 on empty / malformed input rather than throwing — boot must NEVER fail to register the plugin because of a self-check helper |
| T-04-01-02 | Information disclosure | Log::warning context | accept | recommended values (20 / 10M) are public knowledge; current value is the very thing operators need to see in logs to fix the host |
| T-04-01-03 | Denial of Service | self-check runs on every frontend request | mitigate | App::runningInBackend() guard — frontend pageloads execute zero ini_get/Log calls (D-34) |
| T-04-01-04 | Tampering | self-check skipped if Plugin::boot signature changes | accept | boot() return type `: void` enforced by parent PluginBase; PHPStan L10 catches signature drift |
